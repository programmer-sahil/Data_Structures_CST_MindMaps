Key Definitions for Exams - Unit 1

Data Structure: A way of organizing and storing data in a computer so that it can be accessed and modified efficiently.
Abstract Data Type (ADT): A mathematical model that defines data type by its behavior (operations) from the user's point of view, independent of implementation.
Linear Data Structure: Data elements arranged in sequential manner where each element is connected to its previous and next element (e.g., Array, Stack, Queue, Linked List).
Non-Linear Data Structure: Data elements not arranged sequentially; one element can connect to multiple elements forming hierarchical relationships (e.g., Trees, Graphs).
Time Complexity: Amount of time taken by an algorithm to run as a function of input size, typically expressed using Big O notation.
Space Complexity: Amount of memory space required by an algorithm during execution, including input data and auxiliary space.

Key Definitions for Exams - Unit 2

Stack: A linear data structure following LIFO (Last In First Out) principle where insertion and deletion occur at the same end called TOP.
Stack Overflow: Condition when attempting to push an element onto a full stack (in array implementation).
Stack Underflow: Condition when attempting to pop an element from an empty stack.
Queue: A linear data structure following FIFO (First In First Out) principle where insertion occurs at REAR and deletion at FRONT.
Circular Queue: A queue where the last position is connected back to the first position, making efficient use of array space and eliminating false overflow.
Deque (Double-Ended Queue): A queue where insertion and deletion can occur at both front and rear ends.
Recursion: A function that calls itself directly or indirectly, consisting of a base case (termination condition) and recursive case.
Infix Expression: An expression where operator is placed between operands (e.g., A + B).
Postfix Expression: An expression where operator follows the operands (e.g., AB+), eliminates need for parentheses and operator precedence rules.
UNIT 3: Linked Lists

Singly Linked List

Structure:
  • Data field
  • Next pointer
Operations:
  • Insert (Beginning/Middle/End)
  • Delete (First/Middle/Last)
  • Traverse
  • Search

Doubly Linked List

Structure:
  • Data field
  • Next pointer
  • Previous pointer
Advantages:
  • Bidirectional traversal
  • Easier deletion
  • Insert before node
Operations:
  • Insert (3 positions)
  • Delete (3 positions)
  • Forward/Backward traverse

Circular Linked Lists

Singly Circular:
  • Last node → First node
  • No NULL pointer
  • Continuous traversal
Doubly Circular:
  • Both directions circular
  • Last ↔ First connected
Applications:
  • Round Robin Scheduling
  • Music Playlist
  • Browser History

LL Representations

Stack using LL:
  • Push at head: O(1)
  • Pop from head: O(1)
  • Dynamic size
Queue using LL:
  • Enqueue at tail
  • Dequeue from head
  • No overflow

Key Definitions for Exams - Unit 3

Linked List: A linear data structure where elements (nodes) are stored in non-contiguous memory locations, connected using pointers.
Node: Basic unit of a linked list containing two parts: DATA field (stores information) and LINK/NEXT field (stores address of next node).
Singly Linked List: A linked list where each node contains data and a pointer to the next node, traversal possible in only one direction (forward).
Doubly Linked List: A linked list where each node has three fields: data, pointer to next node, and pointer to previous node; allows bidirectional traversal.
Circular Linked List: A linked list where the last node points back to the first node instead of NULL, forming a circular chain.
Header Node: A dummy node at the beginning of a linked list that doesn't store actual data but simplifies operations by eliminating special cases.
Dynamic Memory Allocation: Memory allocated at runtime using functions like malloc(), calloc() in C, allowing flexible memory usage based on requirements.
Self-Referential Structure: A structure that contains a pointer to a structure of the same type, used to implement linked lists and trees.

Key Definitions for Exams - Unit 4

Tree: A non-linear hierarchical data structure consisting of nodes connected by edges, with one node designated as root and no cycles.
Binary Tree: A tree where each node has at most two children, referred to as left child and right child.
Binary Search Tree (BST): A binary tree where for each node: all values in left subtree are less than node's value, and all values in right subtree are greater.
Tree Traversal: Process of visiting all nodes in a tree in a specific order. Types: Inorder (Left-Root-Right), Preorder (Root-Left-Right), Postorder (Left-Right-Root).
Height of Tree: Number of edges in the longest path from root to a leaf node. Height of empty tree is -1, single node is 0.
AVL Tree: A self-balancing BST where the difference of heights of left and right subtrees (balance factor) cannot be more than 1 for all nodes.
B-Tree: A self-balancing search tree where nodes can have multiple keys and more than two children, optimized for systems that read/write large blocks of data.
Graph: A non-linear data structure consisting of vertices (nodes) and edges (connections between nodes). Can be directed or undirected, weighted or unweighted.
Adjacency Matrix: A 2D array of size V×V (V=number of vertices) where matrix[i][j]=1 if edge exists from vertex i to j, else 0. Space: O(V²).
Adjacency List: An array of linked lists where index represents a vertex and the list contains all adjacent vertices. Space: O(V+E).
BFS (Breadth First Search): Graph traversal algorithm that explores all neighbors at present depth before moving to nodes at next depth level. Uses Queue. Time: O(V+E).
DFS (Depth First Search): Graph traversal algorithm that explores as far as possible along each branch before backtracking. Uses Stack/Recursion. Time: O(V+E).
